# cursor-skill

面向 Vue 项目协作场景的指令与规则集合，帮助在代码评审、调试排错和路由检查时保持统一的分析结构与输出质量。

## 项目定位

该仓库用于沉淀可复用的 AI 协作提示模板，当前重点覆盖：
- 页面/组件评审
- 浏览器联动调试
- 接口请求排查
- 路由流程检查
- Vue 项目开发规则约束

## 目录结构

```text
cursor-skill/
├─ commands/                 # 可直接调用的任务指令模板
│  ├─ review-vue-page.md     # Vue 页面/组件审查
│  ├─ debug-with-browser-mcp.md
│  ├─ debug-api-request.md
│  └─ check-router-flow.md
└─ rules/                    # 全局规则与技术栈约束
   ├─ vue-skill.md
   └─ vue3-skill.md
```

## 快速开始

1. 根据当前任务选择对应 `commands` 模板。
2. 将模板内容作为对话起点，补充项目上下文（页面、报错、接口、路由信息等）。
3. 同步参考 `rules` 中的技术栈与输出规范，确保建议与现有项目一致。

## Commands 说明

- `review-vue-page`：用于页面/组件结构化评审，输出整体评价、问题影响与最小改动建议。
- `debug-with-browser-mcp`：结合浏览器侧观察（Console、Network、操作复现）定位前端问题。
- `debug-api-request`：聚焦请求链路与参数/响应一致性，输出“最可能原因 + 验证 + 修复”。
- `check-router-flow`：分析路由配置、守卫与跳转链路，定位 404、白屏、循环跳转等问题。

## Rules 说明

- `vue-skill.md`：通用前端协作规范（语言风格、改动原则、排查结构）。
- `vue3-skill.md`：Vue 3 项目实现偏好与工程约束（Pinia、Router、Axios、UnoCSS 等）。

## 使用建议

- 优先做最小必要改动，避免在修复问题时引入无关重构。
- 输出结论时尽量提供可执行步骤、代码片段或验证方式。
- 对可能影响公共模块的变更，应提前标注风险范围。

## 后续可扩展方向

- 增加按业务域拆分的命令模板（如表单页、列表页、弹窗流）。
