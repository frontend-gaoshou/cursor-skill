# check-router-flow

请检查当前项目中的路由流程与页面跳转逻辑，重点分析以下内容：

1. 当前页面对应的路由配置是否正确
2. path、name、redirect、children 配置是否合理
3. 路由跳转方式是否正确
4. 是否存在路由守卫导致的跳转异常
5. 是否存在登录状态恢复、刷新后丢失状态、动态路由注入时机问题
6. 是否存在权限控制与菜单状态不同步问题
7. 如果出现页面白屏、404、重复跳转、死循环跳转，请分析最可能原因
8. 如果是老项目，优先保持现有路由组织方式；如果是 Vue3 项目，可在保持一致的前提下给出优化建议

输出时请使用：
- 问题现象
- 最可能原因
- 验证方法
- 修复建议
的结构。
