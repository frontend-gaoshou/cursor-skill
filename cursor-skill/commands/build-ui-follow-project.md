# build-ui-follow-project

请先审查当前项目的 UI 实现方式，再开始构建页面，不要直接按你的默认习惯生成代码。

重点检查：
1. 项目使用的样式方案（如 UnoCSS、SCSS、Less、CSS Modules）
2. 是否已有通用组件、页面容器、卡片、按钮、表单等可复用组件
3. 图标如何引入，是否已有 SVG 图标组件或统一图标方案
4. 图片资源如何引入，放在 src/assets、public 还是其他位置
5. 是否有相似页面或可参考的现有实现

实现要求：
- 优先复用项目现有样式体系和组件
- 如果项目使用 UnoCSS，优先使用 UnoCSS
- 如果项目已有 SVG 图标方案，不要重复造图标组件
- 不要随意引入新的样式库或图标库
- 保持与项目现有代码风格、目录结构、命名方式一致

输出时请先简要说明：
- 你发现了什么样式方案
- 你准备复用哪些组件或图标方式
- 然后再开始写页面代码
