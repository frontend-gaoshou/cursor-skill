# debug-with-browser-mcp

请优先结合已配置的 Chrome DevTools MCP 或其他浏览器 MCP 工具调试当前前端问题。

要求：
1. 如果提供了页面地址，先打开页面
2. 观察页面实际表现，必要时截图
3. 检查 console 报错和警告
4. 检查 network 中失败请求、异常状态码、参数和响应
5. 如果能复现用户操作，请实际操作并观察结果
6. 再结合当前项目代码，判断问题属于渲染、状态、路由、请求、样式还是联调问题

输出格式：
- 页面现象
- 浏览器观察结果
- 最可能原因
- 验证方法
- 修复建议

如果 MCP 工具不可用，请明确说明并退回为纯代码分析。
